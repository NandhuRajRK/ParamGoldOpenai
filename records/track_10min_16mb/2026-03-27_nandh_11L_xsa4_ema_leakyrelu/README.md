## Why This Variant

The newer public leaderboard is no longer led by the older `10L Int5MLP + BigramHash` family. The safer attack path is to start from the modern `11L + XSA4 + EMA + Partial RoPE + LN Scale` stack and only add one cheap, high-signal delta from the top run.

This variant does exactly that:
- 11 layers, 512d, 8 heads, 4 KV heads
- 3x MLP (`1536` hidden)
- XSA on the last 4 layers
- Partial RoPE (`16/64`)
- LN scale
- EMA
- SmearGate + BigramHash
- int6 mixed quantization + zstd-22
- **LeakyReLU(0.5)^2** instead of `relu^2`

The goal is to test whether the activation gain from the current public leader transfers cleanly onto the simpler 11-layer EMA/XSA stack before attempting heavier additions such as legal TTT or parameter banking.

## Exact Code Change

```python
# old
x = torch.relu(self.fc(x))
return self.proj(x.square())

# new
x = F.leaky_relu(self.fc(x), negative_slope=0.5)
return self.proj(x.square())
```
